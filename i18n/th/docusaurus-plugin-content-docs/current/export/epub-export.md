# ส่งออก EPUB

EpubCraft ส่งออกหนังสือเป็นไฟล์ EPUB 2.0 มาตรฐาน ที่เข้ากันได้กับเครื่องอ่านและแพลตฟอร์มจัดจำหน่ายเกือบทั้งหมด

## การส่งออก

1. กด `Ctrl+S` หรือคลิกปุ่ม **Export** ในแถบเครื่องมือ
2. เลือก **Standard EPUB**
3. เปิด **Optimize Line Breaking** สำหรับข้อความไทย/เอเชีย (ถ้าต้องการ)
4. คลิก **Export Now**

ไฟล์ EPUB จะดาวน์โหลดอัตโนมัติ

## โครงสร้าง EPUB

EPUB ที่สร้างเป็นไปตามข้อกำหนด EPUB 2.0 (OPF 2.0):

```
book.epub
├── mimetype                          (application/epub+zip)
├── META-INF/
│   └── container.xml                 (ชี้ไปที่ content.opf)
└── OEBPS/
    ├── content.opf                   (เอกสารแพ็คเกจ)
    ├── toc.ncx                       (การนำทาง/สารบัญ)
    ├── Styles/
    │   └── style.css                 (CSS ทั่วไป + กฎ @font-face)
    ├── Fonts/
    │   ├── font1.ttf                 (ฟอนต์ที่ฝัง)
    │   └── font2.otf
    ├── Images/
    │   ├── cover.jpg                 (ภาพปก)
    │   └── image1.png                (รูปภาพในบท)
    └── Text/
        ├── chapter_x00001.xhtml      (บทที่ 1)
        ├── chapter_x00002.xhtml      (บทที่ 2)
        └── ...
```

## สิ่งที่รวมอยู่

### บท
แต่ละบทถูกแปลงเป็น XHTML 1.1 ที่ถูกต้อง ชื่อไฟล์บทตามรูปแบบ `chapter_x00001.xhtml` สำหรับการเรียงลำดับที่ถูกต้อง

### CSS
CSS ทั่วไปรวมอยู่ใน `Styles/style.css` ถ้ามีฟอนต์ที่กำหนดเอง การประกาศ `@font-face` จะถูกเพิ่มอัตโนมัติ

### ฟอนต์
ฟอนต์ที่อัปโหลดทั้งหมดถูกฝังในไดเรกทอรี `Fonts/` และอ้างอิงใน OPF manifest

### รูปภาพ
รูปภาพจากเนื้อหาบทถูกแยกจาก data URI base64 บันทึกเป็นไฟล์แยกใน `Images/` และอ้างอิงด้วยพาธสัมพัทธ์ รูปภาพที่ซ้ำจะถูกตัดทิ้ง

### ปก
ถ้าตั้งค่าภาพปก จะถูกแปลงเป็น JPEG และรวมเป็น `Images/cover.jpg` อ้างอิงใน OPF metadata

### เชิงอรรถ
เครื่องหมายเชิงอรรถ (`[*]`) ถูกแปลงเป็นตัวเลขอ้างอิงพร้อมส่วนเชิงอรรถท้ายบท ดู [เชิงอรรถในการส่งออก](/docs/export/footnotes-in-export)

## OPF Metadata

ไฟล์ `content.opf` มี:

- Dublin Core metadata (ชื่อเรื่อง ผู้แต่ง สำนักพิมพ์ ภาษา คำอธิบาย วันที่)
- ตัวระบุหนังสือเฉพาะ (UUID)
- แท็ก generator: "EpubCraft"
- Manifest รายการไฟล์ทั้งหมดพร้อมประเภท MIME
- Spine กำหนดลำดับการอ่านบท
- การอ้างอิงภาพปก

## สารบัญ (NCX)

ไฟล์ `toc.ncx` ให้การนำทาง:

- แต่ละบทปรากฏเป็นจุดนำทาง
- ชื่อบทใช้เป็นป้าย
- ลำดับการเล่นตรงกับลำดับบท

## การประมวลผล

ขณะส่งออก การประมวลผลต่อไปนี้เกิดขึ้น:

1. เนื้อหา HTML ถูกแปลงเป็น XHTML 1.1 ที่ถูกต้อง
2. รูปภาพถูกแยกและบันทึกเป็นไฟล์
3. ลิงก์ข้ามบทถูกแก้ไข (href mapping)
4. เชิงอรรถถูกแปลงเป็นตัวเลขอ้างอิง
5. กฎ `@font-face` ของฟอนต์ถูกสร้าง
6. ถ้าเปิดใช้ อักขระความกว้างศูนย์ถูกแทรกสำหรับข้อความไทย/เอเชีย
7. แฮช SHA-256 ถูกคำนวณสำหรับการตรวจสอบ

บทถูกประมวลผลเป็นชุดละ 100 บทเพื่อประสิทธิภาพ
