# การจัดการฟอนต์

EpubCraft ให้คุณอัปโหลดฟอนต์ที่กำหนดเองและฝังในไฟล์ EPUB เพื่อให้หนังสือแสดงผลถูกต้องไม่ว่าผู้อ่านจะติดตั้งฟอนต์อะไร

## ตัวจัดการฟอนต์

เปิดตัวจัดการฟอนต์โดยคลิกไอคอน **Type** ในแถบด้านข้าง

### การอัปโหลดฟอนต์

คลิกพื้นที่อัปโหลด (เส้นประ) หรือลากและวางไฟล์ฟอนต์ รูปแบบที่รองรับ:

| รูปแบบ | นามสกุล | ประเภท MIME |
|--------|---------|------------|
| TrueType | `.ttf` | font/ttf |
| OpenType | `.otf` | font/otf |
| Web Open Font Format | `.woff` | font/woff |
| Web Open Font Format 2 | `.woff2` | font/woff2 |

ฟอนต์จะถูกแปลงเป็น base64 และเก็บในหน่วยความจำ

### ชื่อ Font-Family

ชื่อ font-family จะถูกสร้างอัตโนมัติจากชื่อไฟล์โดยลบช่องว่าง ตัวอย่าง:

- `NotoSansThai-Regular.ttf` → font-family: `NotoSansThai-Regular`
- `My Custom Font.otf` → font-family: `MyCustomFont`

ตัวจัดการฟอนต์แสดงกฎ CSS font-family สำหรับแต่ละฟอนต์เพื่อให้คุณคัดลอกไปใส่ใน CSS

### การลบฟอนต์

คลิกไอคอน **ถังขยะ** ข้างฟอนต์เพื่อลบ

## การใช้ฟอนต์ใน CSS

อ้างอิงฟอนต์ที่อัปโหลดใน CSS ทั่วไปด้วยชื่อ font-family:

```css
body {
  font-family: 'NotoSansThai-Regular', sans-serif;
}

h1, h2, h3 {
  font-family: 'MyHeadingFont', serif;
}
```

คุณไม่ต้องเขียนการประกาศ `@font-face` — EpubCraft สร้างให้อัตโนมัติขณะส่งออก

## การฝังฟอนต์ใน EPUB

เมื่อส่งออกเป็น EPUB ฟอนต์ที่อัปโหลดทั้งหมดจะ:

1. รวมอยู่ในไดเรกทอรี `OEBPS/Fonts/`
2. อ้างอิงใน OPF manifest
3. จับคู่กับการประกาศ `@font-face` ที่สร้างอัตโนมัติใน CSS

เพื่อให้ฟอนต์เดินทางไปกับไฟล์ EPUB และแสดงถูกต้องบนเครื่องอ่านที่รองรับฟอนต์ฝัง

## ฟอนต์เทมเพลต

เทมเพลต CSS ที่ต้องการฟอนต์เฉพาะ (เช่น Noto Sans Thai หรือ Sarabun) จะดาวน์โหลดและเพิ่มฟอนต์ในคลังอัตโนมัติเมื่อใช้ ฟอนต์เหล่านี้ทำงานเหมือนฟอนต์ที่อัปโหลดเอง
