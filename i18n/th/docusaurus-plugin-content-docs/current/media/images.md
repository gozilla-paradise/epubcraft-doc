# รูปภาพ

EpubCraft รองรับการฝังรูปภาพโดยตรงในบทและผลลัพธ์ EPUB

## การแทรกรูปภาพ

### จากแถบเครื่องมือตัวแก้ไข

1. วางเคอร์เซอร์ที่ตำแหน่งที่ต้องการ
2. คลิกปุ่ม **รูปภาพ** ในแถบเครื่องมือจัดรูปแบบ
3. เลือกไฟล์ภาพจากคอมพิวเตอร์

รูปภาพจะถูกแปลงเป็น base64 และแทรกเป็น data URI:

```html
<img src="data:image/png;base64,iVBOR..." alt="image" />
```

### รูปแบบรูปภาพที่รองรับ

| รูปแบบ | นามสกุล | ประเภท MIME |
|--------|---------|------------|
| JPEG | `.jpg`, `.jpeg` | image/jpeg |
| PNG | `.png` | image/png |
| GIF | `.gif` | image/gif |
| SVG | `.svg` | image/svg+xml |
| WebP | `.webp` | image/webp |

## การเก็บรูปภาพ

รูปภาพจะถูกเก็บเป็น data URI แบบ base64 ในเนื้อหา HTML ของบทโดยตรง หมายความว่า:

- รูปภาพเป็นอิสระ — ไม่มีการอ้างอิงไฟล์ภายนอก
- รูปภาพคงอยู่เมื่อบทถูกจัดเรียงหรือทำซ้ำ
- รูปภาพรวมอยู่เมื่อส่งออกเป็น EPUB หรือ ZIP

## รูปภาพในการส่งออก EPUB

ขณะส่งออก EPUB รูปภาพที่ฝังในเนื้อหาบทจะ:

1. ถูกแยกจาก data URI แบบ base64
2. บันทึกเป็นไฟล์แยกใน `OEBPS/Images/`
3. อ้างอิงใน OPF manifest พร้อมประเภท MIME ที่ถูกต้อง
4. แอตทริบิวต์ `src` ของ `<img>` ถูกเขียนใหม่เป็นพาธสัมพัทธ์

รูปภาพที่ซ้ำกัน (ข้อมูลเดียวกัน) จะถูกตรวจจับและรวมเป็นหนึ่งเดียว

## แนวทางปฏิบัติที่ดี

- **ขนาด**: เก็บรูปภาพไม่เกิน 1MB ต่อรูป รูปใหญ่เพิ่มขนาดไฟล์ EPUB
- **ความละเอียด**: 72–150 DPI เพียงพอสำหรับเครื่องอ่านส่วนใหญ่
- **รูปแบบ**: JPEG สำหรับภาพถ่าย PNG สำหรับกราฟิกที่มีพื้นโปร่งใส
- **ความกว้าง**: ตั้ง `max-width: 100%` ใน CSS เพื่อป้องกันรูปล้นหน้า:

```css
img {
  max-width: 100%;
  height: auto;
}
```

## รูปภาพจาก EPUB ที่นำเข้า

เมื่อนำเข้าไฟล์ EPUB รูปภาพทั้งหมดจะถูกแยกและแปลงเป็น data URI แบบ base64 พาธรูปภาพใน HTML (`../Images/filename.jpg`) จะถูกแทนที่ด้วยข้อมูลที่ฝังโดยอัตโนมัติ
