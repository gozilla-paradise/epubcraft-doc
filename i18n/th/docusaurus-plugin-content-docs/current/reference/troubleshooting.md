# การแก้ไขปัญหา

ปัญหาทั่วไปและวิธีแก้เมื่อใช้ EpubCraft

## ปัญหาการนำเข้า

### นำเข้า EPUB แล้วบทว่าง
EPUB อาจใช้โครงสร้างที่ไม่มาตรฐาน EpubCraft อ่าน OPF spine เพื่อกำหนดลำดับบท ถ้าบทว่าง:
- EPUB อาจมีการป้องกัน DRM (ไม่รองรับ)
- ไฟล์เนื้อหาอาจใช้ชื่อหรือพาธที่ไม่มาตรฐาน
- ลองส่งออก EPUB จากเครื่องมืออื่นก่อน

### นำเข้า ZIP แล้วไม่กู้คืนฟอนต์
ตรวจสอบให้แน่ใจว่าไฟล์ฟอนต์อยู่ที่ระดับรากของ ZIP ไม่ใช่ในไดเรกทอรีย่อย รูปแบบที่รองรับ: `.ttf`, `.otf`, `.woff`, `.woff2`

### การแปลง Markdown ไม่สมบูรณ์
EpubCraft รองรับ Markdown พื้นฐาน: `**ตัวหนา**`, `*ตัวหนา*`, `__ตัวเอียง__`, `_ตัวเอียง_` และหัวข้อ Markdown ขั้นสูง (ตาราง บล็อกโค้ด รายการซ้อน) ไม่ถูกแปลงเต็ม แปลงเป็น HTML ภายนอกก่อนสำหรับ Markdown ที่ซับซ้อน

## ปัญหาการส่งออก

### ข้อผิดพลาดการตรวจสอบ EPUB
ถ้า EPUB ไม่ผ่านการตรวจสอบ:
- ตรวจสอบแท็ก HTML ที่ไม่ได้ปิดในบท
- ตรวจสอบว่ารูปภาพทั้งหมดมีข้อมูล base64 ที่ถูกต้อง
- ลบ JavaScript หรือเนื้อหาที่ไม่เข้ากับ EPUB
- ใช้ปุ่ม **Format** เพื่อแก้ไขการย่อหน้า HTML อัตโนมัติ

### ขนาดไฟล์ EPUB ใหญ่
- ปรับแต่งรูปภาพก่อนอัปโหลด (ลดความละเอียดเป็น 72–150 DPI)
- ใช้ JPEG สำหรับภาพถ่ายแทน PNG
- ลบฟอนต์ที่ไม่ได้ใช้จากตัวจัดการฟอนต์
- หลีกเลี่ยงรูปภาพ base64 ขนาดใหญ่มากในเนื้อหาบท

### ข้อความไทยไม่ตัดบรรทัดถูกต้อง
เปิด **Optimize Line Breaking** ในหน้าต่างส่งออก เพิ่มอักขระความกว้างศูนย์ที่ขอบเขตคำ ต้องเชื่อมต่ออินเทอร์เน็ตสำหรับบริการตัดคำ

## ปัญหาตัวแก้ไข

### ไฮไลท์ซินแท็กซ์ไม่อัปเดต
ตัวแก้ไขใช้การเรนเดอร์แบบ debounce (700ms) ถ้าไฮไลท์ค้าง:
- คลิกนอกตัวแก้ไขแล้วกลับเข้ามา
- ลองสลับการตัดบรรทัด
- รีเฟรชหน้า (บันทึกงานก่อนผ่านการส่งออก ZIP)

### พรีวิวไม่แสดงการเปลี่ยนแปลง
- ตรวจสอบว่าอยู่ในโหมด **แบ่งหน้าจอ** หรือ **พรีวิว**
- ตรวจสอบว่าเลือกบทที่ถูกต้องสำหรับพรีวิว (ในโหมด CSS)
- ตรวจสอบว่า CSS ไม่มีข้อผิดพลาดซินแท็กซ์ที่ป้องกันการเรนเดอร์

### เลิกทำไม่ทำงานตามที่คาดหวัง
ตัวแก้ไขจับ snapshot ประวัติทุก 700ms การแก้ไขรวดเร็วภายในช่วงนั้นจะถูกรวมเป็นขั้นตอนเลิกทำเดียว ใช้ปุ่มเลิกทำในแถบเครื่องมือตัวแก้ไข (ไม่ใช่เลิกทำของเบราว์เซอร์) สำหรับผลลัพธ์ที่ดีที่สุด

## ปัญหาฟอนต์

### ฟอนต์ที่กำหนดเองไม่แสดงในพรีวิว
- ตรวจสอบว่าฟอนต์อัปโหลดสำเร็จ (ดูตัวจัดการฟอนต์)
- ตรวจสอบว่า CSS อ้างอิงชื่อ font-family ที่ถูกต้อง (ลบช่องว่างจากชื่อไฟล์)
- ตรวจสอบคอนโซลเบราว์เซอร์สำหรับข้อผิดพลาดการโหลดฟอนต์

### ฟอนต์ไม่ปรากฏใน EPUB
ฟอนต์ที่อัปโหลดทั้งหมดจะถูกฝังอัตโนมัติ ถ้าฟอนต์ไม่ทำงานใน EPUB ที่ส่งออก:
- เครื่องอ่านบางตัวไม่รองรับฟอนต์ทุกรูปแบบ — ลองใช้ TTF หรือ OTF
- ตรวจสอบว่าชื่อ font-family `@font-face` ใน CSS ตรงกับฟอนต์ที่อัปโหลด

## ปัญหา AI

### ข้อผิดพลาด "API_KEY_MISSING"
ตั้งค่า Gemini API key:
1. คลิกไอคอน **กุญแจ** ในแถบด้านข้าง
2. ป้อน API key ที่ถูกต้องจาก [Google AI Studio](https://aistudio.google.com/app/apikey)
3. คลิก **Save Key**

### AI ไม่สร้างผลลัพธ์
- ตรวจสอบการเชื่อมต่ออินเทอร์เน็ต
- ตรวจสอบว่า API key ถูกต้องและมีโควต้าเหลือ
- ลองคำสั่งที่สั้นและเฉพาะเจาะจงมากขึ้น
- โมเดลอาจมีอัตราจำกัด — รอสักครู่แล้วลองอีกครั้ง

## ปัญหาทั่วไป

### แอปช้าเมื่อมีหลายบท
EpubCraft ประมวลผลบทในหน่วยความจำ หนังสือขนาดใหญ่มาก (100+ บท หรือรูปภาพใหญ่หลายรูป) อาจทำให้ประสิทธิภาพลดลง พิจารณา:
- แบ่งโปรเจกต์ใหญ่มากเป็นหลายหนังสือ
- ปรับแต่งขนาดรูปภาพ
- ปิดแท็บเบราว์เซอร์อื่นเพื่อเพิ่มหน่วยความจำ

### ข้อมูลหายหลังรีเฟรชเบราว์เซอร์
EpubCraft ทำงานทั้งหมดในหน่วยความจำและไม่บันทึกอัตโนมัติ ส่งออกโปรเจกต์เป็น ZIP เสมอก่อนปิดแท็บเบราว์เซอร์ ไม่มีทางกู้คืนงานที่ไม่ได้บันทึกหลังรีเฟรชหน้า

### ปัญหาเลย์เอาต์มือถือ
ฟีเจอร์บางอย่าง (แบ่งหน้าจอ ลากและวาง) มีข้อจำกัดบนมือถือ:
- ใช้โหมดโค้ดหรือพรีวิว (แบ่งหน้าจอซ่อนบนมือถือ)
- การจัดเรียงบทด้วยลากอาจไม่แม่นยำบนหน้าจอสัมผัส
- แถบด้านข้างเปิดเป็นลิ้นชักเลื่อนเข้า — แตะนอกพื้นที่เพื่อปิด
